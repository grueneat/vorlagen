01Strategy

Strategy summary

The swap is mechanical and verified end-to-end on the live container.Patched postkarte build.py to Barlow, rendered via the sanctioned xvfb-run … scribus … _export_pdf.py pipeline; pdffonts shows all three Barlow weights embedded with zero DejaVu fallback.
Scribus names the four faces exactly Barlow Semi Condensed Regular / Bold / Black / ExtraBold.Confirmed via scribus.getFontNames(). Scribus reads the TTF name table directly, so no fontconfig alias is needed (unlike Vollkorn).
Fonts live in three layers, all must change.16 per-template build.py (7 distinct non-Barlow strings, ~1100 refs), 3 top-level *-original.sla, plus two library defaults the prior map missed: brand.py deffont and shared/ci.yml::fonts.
The hard part is baselines, not the swap.Barlow's narrower metric shifts line-wrap/centering, invalidating the Gotham-tuned TOLERANCES.yml. New baselines must be promoted from fresh previews after page-by-page review of all ~72 pages, then tolerances re-derived.
CI never renders.It runs the SHA-based stale-previews gate (auto-updated by render-gallery) plus a non-blocking brand validator — all baselines/previews are produced locally and committed. sla_diff will NOT block: all 3 original_sla templates are sla_diff_strict: false.

03Decisions

Key decisions & options

Provision Barlow via fontconfig — no aliaschosen
Scribus resolves "Barlow Semi Condensed Black/Bold/Regular" natively from the TTF name table (verified by a real render). A fontconfig family-alias (as Vollkorn needs) is unnecessary. Sanity-check on fc-match "Barlow Semi Condensed" + a face count, NOT on the per-weight strings (Regular/Bold fall back to DejaVu at fc-match level but render fine).
Edit build.py, not template.slachosen
Every template.sla regenerates from build.py; direct SLA edits are overwritten on the next render. Only the 3 hand-authored top-level *-original.sla oracles get direct FONT/DFONT edits (incl. the lone Gotham Narrow Ultra Italic in postkarte original).
Re-derive TOLERANCES.yml empiricallyopen
The existing per-template drift budgets encode Gotham-vs-InDesign line-wrap deltas (some up to 260 issues). Barlow wraps differently, so they no longer apply. Render with --skip-visual-diff, promote new baselines, then re-derive budgets from the new render's audits. Exact values only knowable post-render (LOW confidence).
Don't hand-roll the render / baseline / stale plumbingnote
Use render_sla_to_pdf / bin/render-gallery (auto-updates meta SHAs), tools/font_audit.py (DejaVu detection), and the documented rebaseline recipe in docs/render-fidelity.md:118.
Vollkorn-italic accent → upright Barlow Blackrisk
No Barlow italic TTF exists locally. Decision 1 accepts upright Barlow Black for the 55 Vollkorn-italic refs. Only pull a Barlow Italic (Google Fonts, OFL) if visual review deems a slanted accent essential — flag to user, do not silently add.
